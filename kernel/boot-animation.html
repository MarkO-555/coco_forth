CoCo Kernel Boot Sequence

Two kernel build profiles, two boot sequences. ROM mode (default, make) keeps BASIC ROMs alive and runs the kernel at $2000. All-RAM mode (make allram) pages out the ROMs and relocates the kernel to $E000 via a bootstrap copy.

Power On
BASIC + DECB own everything
SAM maps three 8K ROMs to upper 32K — Extended Color BASIC ($8000), Color BASIC ($A000), and the cartridge slot ($C000). Lower 32K is RAM, carved up by BASIC + DECB:

BASIC system vars at $0000-$01FF, graphics + line buffer at $0200-$03FF, default text VRAM at $0400-$05FF, DECB disk system at $0600-$0DFF, BASIC program area at $0E00-$7DFF, and the 6809 hardware stack at $7E00-$7FFF (S grows down from $7FFF).

I/O sits at $FF00-$FFFF (PIA, SAM, vectors).
Step 1
LOADM drops kernel + app into RAM
XRoar (or BASIC at the prompt) loads the DECB binary into lower 32K. LOADM can write anywhere from $0000 up to $7FFF (BASIC ROMs cover $8000+):

Kernel at $2000 (~3.7K, includes baked-in font-data and sin-data tables), application + variables at $3000-$7FFF (up to ~20K available).

No bootstrap, no staging, no copy — the kernel was assembled directly at $2000.
Step 2
JMP START — kernel runs alongside ROM
DECB exec is START. The kernel initialises DP variables at $0050, the PIA, VDG text VRAM at $0400, sets up stacks (S=$8000, U=$7E00), and enters the Forth thread at $3000 via NEXT.

The app calls init-font (copies kernel font-data to $5800), init-sin (fills the sin table at $7800), and rg-init (points VRAM to $0600). Calling basic-cold (JMP $A027) returns to the OK prompt — only possible because the BASIC ROMs are still alive.
Power On
BASIC + DECB own everything
Identical starting state to ROM mode. SAM maps three 8K ROMs to upper 32K (ECB, CB, cartridge); lower 32K is divided between BASIC system + variables, default text VRAM at $0400-$05FF, DECB disk buffers at $0600-$0DFF, BASIC's program area, and the hardware stack near $7FFF.
Step 1
LOADM stages everything in lower 32K
LOADM reaches exactly to $7FFF (BASIC ROMs cover $8000+), so fc.py remaps every record at $E000+ to a staging address in low RAM:

DP vars at $0050, bootstrap at $0E00, staged kernel at $1000, application at $2000-$7FFF (24K).

The kernel was assembled at $E000; it'll be copied to its real home in step 3. Font glyphs at $9000 and the sin table at $86CC are runtime-initialised by the kernel after all-RAM is engaged — they aren't LOADM records.
Step 2
Bootstrap enables all-RAM
DECB exec is $0E00 (bootstrap).

ORCC #$50 — mask interrupts
STA $FFDF — set SAM TY: all-RAM

ROM disappears. $8000-$FEFF is now writable RAM (font glyphs at $9000 get copied into place by the kernel, runtime data zone $8000-$DDFF is free for buffers and tables).
Step 3
Copy $1000 → $E000
Bootstrap copies the staged kernel to its final location:

LDX #$1000 — source
LDY #$E000 — destination
LDD ,X++ / STD ,Y++ word-copy loop
CMPY #KERN_END / BLO

~3.7 KB copied in ~960 iterations.
Step 4
JMP START — kernel runs at $E000
JMP START — the kernel initialises PIA, sets up stacks (S=$E000, U=$DE00), and enters the Forth thread at $2000 via NEXT.

The app calls rg-init, which points VRAM to $0600 — harmlessly overwriting the dead bootstrap and staged kernel. basic-cold is unsafe in this mode (BASIC ROMs are gone); use bye instead.
