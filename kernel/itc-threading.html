Indirect Threaded Code, one micro-step at a time

How Bare Naked Forth actually runs a word — the double indirection, the two stacks, LIT / NEXT / DOCOL / EXIT / HALT

A Forth word is not a subroutine you JSR to. A colon word is a thread: a list of CFAs (code-field addresses); a primitive is machine code fronted by a CFA. The inner interpreter walks the thread with NEXT, and each entry is reached by a double indirection: thread cell → CFA → machine code. Registers: X = IP, U = data stack, S = return stack, Y = W (scratch, holds the CFA in hand). Below we run MAIN = LIT 5  DOUBLE  HALT where : DOUBLE DUP + ; — push 5, double it to 10, then halt. Press Step.

NEXT = LDY ,X++ (fetch CFA into W, bump IP) then JMP [,Y] (jump through the CFA). Every primitive ends with it.
LIT = LDD ,X++ (read the inline literal, step IP past it) / STD ,--U / NEXT — this is how constants ride inside the thread.
DOCOL (a colon word’s code) = PSHS X (save caller IP) / LEAX 2,Y (IP → body) / NEXT. EXIT = PULS X (restore caller IP) / NEXT. HALT = BRA HALT — branches to itself forever; the one primitive that does not do NEXT.
Amber = inline data (not a CFA); yellow = cell just read; green = where IP points; pink arrow = the JMP [,Y] indirection.
